# 📱 숏폼 스트리밍 데모

## 1. 개요

이 프로젝트는 **UI 완성도보다 실무 역량 증명에 초점을 둔 최소 단위 숏폼 스트리밍 앱**입니다.

다음과 같은 실제 서비스 환경 문제를 다룹니다:

* 빠른 스크롤 환경에서의 안정적인 영상 재생
* 플레이어 라이프사이클 관리
* 상태 흐름 일관성 유지
* 다음 콘텐츠를 위한 사전 로딩 전략

## 3. 핵심 문제와 해결 전략

> 실제 스트리밍 서비스 환경에서 발생할 수 있는 문제를 가정하고, 성능 및 안정성 관점에서 해결 전략을 설계했습니다.

### 문제 1: 플레이어 인스턴스 과다 생성으로 인한 성능 저하

**문제 상황**

* 리스트 아이템 단위로 플레이어를 생성할 경우
* 메모리 사용량 증가 및 GC 빈도 상승
* 초기화 비용으로 인한 첫 프레임 지연 발생

**해결 방법**

* 단일 ExoPlayer 인스턴스를 유지하고 View에 재바인딩하는 구조 채택
* 플레이어와 UI를 분리하여 재사용 가능하도록 PlayerController 레이어 구성

**결과**

* 플레이어 생성/해제 비용 제거
* 스크롤 시 안정적인 프레임 유지

**트레이드오프**

* 아이템 전환 시 플레이어 attach/detach 타이밍 관리 필요
* 라이프사이클 복잡도 증가

---

### 문제 2: 빠른 스크롤 환경에서의 재생 타이밍 불안정

**문제 상황**

* 사용자가 빠르게 스크롤할 경우
* 여러 아이템이 동시에 partially visible 상태가 됨
* 잘못된 타이밍에 재생 트리거 발생 → UX 저하

**해결 방법**

* LazyListState 기반으로 visible item 중 노출 비율이 가장 높은 아이템을 계산
* 스크롤 idle 상태에서만 재생 트리거 발생
* 이벤트 디바운싱 적용으로 불필요한 재생 요청 차단

**결과**

* 의도하지 않은 재생 방지
* 사용자 시선 기준의 자연스러운 재생 흐름 확보

**트레이드오프**

* 즉시 재생 대비 약간의 지연 발생 가능

---

### 문제 3: 다음 콘텐츠 재생 시 초기 버퍼링 지연

**문제 상황**

* 영상 전환 시 네트워크 요청 및 디코딩 준비로 인해 지연 발생
* 짧은 콘텐츠 특성상 작은 지연도 UX에 큰 영향

**해결 방법**

* 현재 재생 중인 아이템 기준으로 다음 아이템을 사전 prepare
* ExoPlayer의 MediaItem 큐를 활용한 preloading 전략 적용
* 최소 buffer 유지 정책 설정

**결과**

* 영상 전환 시 체감 지연 감소
* 연속 재생 경험 개선

**트레이드오프**

* 추가 네트워크 사용량 증가
* 메모리 점유 증가 가능성

---

### 문제 4: 앱 라이프사이클 변화에 따른 플레이어 상태 불일치

**문제 상황**

* 백그라운드 전환, 화면 회전 등에서 플레이어 상태와 UI 상태 불일치 발생 가능

**해결 방법**

* ViewModel을 단일 상태 소스로 사용하여 재생 상태 관리
* Lifecycle 이벤트를 PlayerController에 명시적으로 전달
* 상태 복구 시 UI와 Player 상태를 동기화

**결과**

* 라이프사이클 변화에도 일관된 재생 상태 유지

**트레이드오프**

* 상태 관리 로직 증가

## 4. 아키텍처

### 패턴

* MVI (단방향 데이터 흐름)

### 레이어 구성

* UI (Compose)
* ViewModel (상태 관리 및 이벤트 처리)
* PlayerController (플레이어 추상화)

### 상태 구성

* UI State: 현재 인덱스, 재생 여부
* Player State: buffering, ready, playing
