# 📱 숏폼 스트리밍 데모

## 1. 개요

이 프로젝트는 **UI 완성도보다 실무 역량 증명에 초점을 둔 최소 단위 숏폼 스트리밍 앱**입니다.

다음과 같은 실제 서비스 환경 문제를 다룹니다:

* 빠른 스크롤 환경에서의 안정적인 영상 재생
* 플레이어 라이프사이클 관리
* 상태 흐름 일관성 유지
* 다음 콘텐츠를 위한 사전 로딩 전략

---

## 2. 목표

### 주요 목표

* 스크롤 시 끊김 없는 영상 재생
* 즉시 재생 가능한 UX
* 안정적인 상태 관리 구조
* 예측 가능한 UI 상태 흐름

### 비목표

* 화려한 UI/애니메이션
* 서버 연동
* 완성형 서비스 구현

---

## 3. 핵심 문제와 해결 전략

### 문제 1: 플레이어 반복 생성으로 인한 성능 저하

**문제 상황**

* 리스트 아이템마다 플레이어를 생성할 경우
* 메모리 사용 증가 및 초기화 비용 발생

**해결 방법**

* 단일 플레이어 인스턴스 유지
* 현재 활성 아이템에 재바인딩

**트레이드오프**

* 재바인딩 타이밍 관리 필요
* 라이프사이클 복잡도 증가

---

### 문제 2: 스크롤 시 재생 지연 및 끊김

**문제 상황**

* 빠른 스크롤 시 어떤 아이템을 재생해야 할지 불명확
* 잘못된 타이밍에 재생 시작

**해결 방법**

* 가장 많이 노출된 아이템 기준으로 재생
* 스크롤 안정화 이후 재생 트리거

**트레이드오프**

* 약간의 지연 vs 안정적인 UX

---

### 문제 3: 다음 영상 재생 지연 (버퍼링)

**문제 상황**

* 다음 콘텐츠 로딩 지연으로 UX 저하

**해결 방법**

* 다음 영상 사전 로딩
* 최소 버퍼 유지

**트레이드오프**

* 네트워크 및 메모리 사용 증가

---

## 4. 아키텍처

### 패턴

* MVI (단방향 데이터 흐름)

### 레이어 구성

* UI (Compose)
* ViewModel (상태 관리 및 이벤트 처리)
* PlayerController (플레이어 추상화)

### 상태 구성

* UI State: 현재 인덱스, 재생 여부
* Player State: buffering, ready, playing

---

## 5. 기술 스택

* Kotlin
* Jetpack Compose
* ViewModel
* Coroutine / Flow
* ExoPlayer

## 6. 주요 기능

### 1. 세로 피드

* LazyColumn 기반 스크롤
* 단일 활성 아이템 구조

### 2. 자동 재생

* 화면에 진입 시 자동 재생
* 이탈 시 자동 정지

### 3. 라이프사이클 대응

* 백그라운드/포그라운드 전환 처리
* 안전한 리소스 해제

### 4. 프리로딩

* 다음 영상 사전 준비
* 재생 지연 최소화
